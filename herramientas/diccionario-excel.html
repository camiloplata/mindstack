BUSCARV
Buscar Vertical
Busca un valor en la primera columna de un rango y devuelve un valor de la misma fila en otra columna.
Sintaxis
=BUSCARV(valor_buscado, rango_tabla, núm_columna, [coincidencia])
// coincidencia: 0 = exacta (usa siempre 0 en finanzas)
Cómo funciona
Toma el valor_buscado, lo busca de arriba hacia abajo en la primera columna del rango_tabla, y cuando lo encuentra devuelve el valor de la columna número núm_columna en esa misma fila. Con 0 (o FALSO) exige coincidencia exacta — obligatorio para clasificar cuentas contables donde un número incorrecto arruina el análisis.
Ejemplo del curso — Clase 3 · Diccionario Financiero
Casa Palma · Diccionario Financiero
=BUSCARV(A2, DiccionarioFinanciero!$A:$D, 2, 0)
En el ERP de Casa Palma llegan 300+ cuentas contables crudas (ej: cuenta 4135). Esta fórmula busca ese código en el Diccionario y trae automáticamente el Rubro correspondiente (Ingresos, Costo de Ventas, etc.), clasificando miles de registros en segundos sin hacerlo manualmente.
Errores comunes
#N/AEl valor buscado no existe en la primera columna. Causa frecuente: el código del ERP tiene espacios o formato diferente al diccionario. Solución: usar ESPACIOS() o VALOR() para limpiar primero.
#REF!El núm_columna es mayor al número de columnas del rango. Revisá que el rango incluya la columna que querés devolver.
Valor incorrectoOlvidaste poner el 0 al final y Excel hace coincidencia aproximada. Siempre usá 0 en modelos financieros.

BUSCARX
Buscar X (versión moderna)
Versión avanzada de BUSCARV: busca en cualquier dirección, devuelve rangos enteros y maneja errores nativamente.
Sintaxis
=BUSCARX(valor_buscado, rango_búsqueda, rango_retorno, [si_no_encontrado], [modo_coincidencia])
// si_no_encontrado: texto o valor si no hay resultado (evita #N/A)
Cómo funciona
A diferencia de BUSCARV, el rango de búsqueda y el rango de retorno son independientes — podés buscar en cualquier columna y devolver cualquier otra. Esto elimina el problema de "contar columnas" de BUSCARV. Además el parámetro si_no_encontrado permite devolver 0 o un texto en vez de mostrar #N/A, limpiando el modelo automáticamente.
Ejemplo del curso — Clase 3 · Diccionario Financiero
Casa Palma · Clasificación automática
=BUSCARX(A2, DiccionarioFinanciero[Cuenta], DiccionarioFinanciero[Subrubro], "Sin clasificar")
Busca el código de cuenta (ej: 4135) en la columna Cuenta del diccionario y devuelve el Subrubro (ej: "Ventas Minoristas"). Si la cuenta no existe en el diccionario, devuelve "Sin clasificar" en lugar de un error — esto es clave para detectar cuentas nuevas del ERP que aún no están clasificadas.
Errores comunes
#N/ASolo ocurre si olvidás el parámetro si_no_encontrado. Siempre completalo en modelos financieros.
No disponibleBUSCARX requiere Excel 365 o Excel 2019+. En versiones anteriores usá BUSCARV.

SI / IF
Condicional Simple
Evalúa una condición y devuelve un valor si es verdadera, otro si es falsa.
Sintaxis
=SI(condición, valor_si_verdadero, valor_si_falso)
// En inglés: =IF(...) · Se pueden anidar hasta 64 niveles
Cómo funciona
Evalúa la condición (puede ser una comparación, fórmula o celda). Si el resultado es VERDADERO devuelve el primer valor; si es FALSO, el segundo. Se anidan varios SI para manejar múltiples casos, como asignar semáforos de alerta (Verde / Amarillo / Rojo) a KPIs financieros según rangos.
Ejemplo del curso — Clase 10 · Capacidad de Préstamo
Casa Palma · KPIs de Apalancamiento
=SI(B24<2.5, "Verde", SI(B24<3.5, "Amarillo", "Rojo"))
Evalúa el ratio Deuda/EBITDA de Casa Palma (celda B24). Si es menor a 2.5x → Verde (negocio sano para banco). Si está entre 2.5x y 3.5x → Amarillo (zona de atención). Si supera 3.5x → Rojo (riesgo de rechazo de crédito). Este semáforo es la base del dashboard de apalancamiento de la Clase 10.
Errores comunes
VERDADERO/FALSO inesperadoOlvidaste las comillas en el texto: =SI(A1>0, Verde, Rojo) devuelve error porque Excel cree que "Verde" y "Rojo" son rangos. Correcto: =SI(A1>0, "Verde", "Rojo").
Lógica invertidaConfundir el orden de los parámetros (verdadero/falso). Siempre leer en voz alta: "Si esto... entonces esto... si no, esto otro".

SI.ERROR
Manejo de Errores
Atrapa cualquier error de una fórmula y devuelve un valor alternativo en su lugar.
Sintaxis
=SI.ERROR(fórmula, valor_si_error)
// En inglés: =IFERROR(...) · Captura #N/A, #DIV/0!, #REF!, #VALOR!, etc.
Cómo funciona
Ejecuta la fórmula normalmente. Si produce cualquier tipo de error (#N/A, #DIV/0!, #REF!, etc.), en lugar de mostrar el error devuelve el valor_si_error que definas (generalmente 0, un guión "-" o texto explicativo). Imprescindible en dashboards financieros donde una división por cero rompería toda la visualización.
Ejemplo del curso — Clase 7 · Rentabilidad y KPIs
Casa Palma · Dashboard KPI · Margen Bruto
=SI.ERROR(B9/B6, 0)
Calcula el margen bruto de una sucursal de Casa Palma (utilidad bruta / ingresos). Si los ingresos de ese mes son 0 (sucursal sin ventas o mes no iniciado), en vez de romper el dashboard con #DIV/0! devuelve 0. Esto permite que el gráfico del dashboard siga funcionando correctamente para todos los meses del año.
Errores comunes
Ocultar errores realesSI.ERROR aplicado a una BUSCARV que da #N/A puede esconder cuentas sin clasificar. En el diccionario financiero es mejor dejar que el error sea visible para auditar cuentas faltantes.
0 vs. blancoDevolver 0 en vez de "" puede distorsionar promedios. Para gráficos, devolver "" (vacío) evita que se trace la línea en ese punto.

Y / AND
Lógica Y — Todas deben cumplirse
Devuelve VERDADERO solo si TODAS las condiciones evaluadas son verdaderas.
Sintaxis
=Y(condición1, condición2, [condición3...])
// En inglés: =AND(...) · Se usa casi siempre dentro de un SI()
Cómo funciona
Evalúa cada condición en paralelo. Si todas son verdaderas, devuelve VERDADERO. Si al menos una es falsa, devuelve FALSO. En modelos financieros se usa para verificar que múltiples checks pasen simultáneamente — por ejemplo, que el balance esté cuadrado Y que el flujo de caja sea positivo Y que los márgenes estén dentro del presupuesto.
Ejemplo del curso — Clase 19 · Auditoría del Modelo
Casa Palma · Hoja Checks · Verificación global
=SI(Y(CONTAR.SI(B6:N6,"ERROR")=0, CONTAR.SI(B12:M12,"ERROR")=0, CONTAR.SI(B18:M18,"ERROR")=0), "✅ MODELO OK — TODAS LAS VERIFICACIONES PASAN", "❌ HAY ERRORES — REVISAR ARRIBA")
Esta es la celda de verificación global del modelo de Casa Palma. Solo muestra "✅ MODELO OK" si las tres filas de checks (P&L, Balance y Flujo de Caja) no tienen ninguna celda con "ERROR". Si cualquier check falla, muestra la alerta en rojo. Un solo punto de control para auditar todo el modelo.
Errores comunes
Confundir Y con OY requiere que TODAS sean verdaderas; O requiere que al menos UNA sea verdadera. En auditoría de modelos usamos Y porque queremos que todos los checks pasen.
📚 Clase 19 · Auditoría modelo
O / OR
Lógica O — Al menos una debe cumplirse
Devuelve VERDADERO si AL MENOS UNA de las condiciones evaluadas es verdadera.
Sintaxis
=O(condición1, condición2, [condición3...])
// En inglés: =OR(...) · Siempre dentro de un SI() en la práctica
Cómo funciona
Complemento de Y(). Devuelve VERDADERO si cualquiera de las condiciones se cumple. En el diccionario financiero se usa para clasificar grupos de cuentas contables: "si el primer dígito es 1, 2 o 3 → Balance; si es 4, 5, 6, 7 u 8 → P&L".
Ejemplo del curso — Clase 3 · Diccionario Financiero
Casa Palma · Ejercicio Diccionario · Clasificación de Estado Financiero
=SI(O(C8="1", C8="2", C8="3"), "Balance", "P&L")
El primer dígito de cada cuenta contable indica su tipo. Si ese dígito es 1 (Activos), 2 (Pasivos) o 3 (Patrimonio), la cuenta va al Balance General. Cualquier otro (4=Ingresos, 5=Costos, 6-8=Gastos) va al P&L. Esta lógica clasifica automáticamente todas las cuentas del ERP sin revisarlas una por una.
Errores comunes
Tipo de datoSi el código de cuenta viene como número (1) y lo comparás con texto ("1"), O() puede fallar. Verificá que ambos lados de la comparación sean del mismo tipo.

SUMA
Suma de Rangos
Suma todos los valores numéricos de un rango de celdas.
Sintaxis
=SUMA(rango1, [rango2], [...])
// En inglés: =SUM(...) · Ignora texto y celdas vacías automáticamente
Cómo funciona
Suma todos los valores numéricos del rango. Las celdas con texto o vacías se ignoran. En modelos financieros se usa en cada total de fila o columna del P&L y Balance — es la base de todos los subtotales (Utilidad Bruta, EBITDA, Utilidad Neta).
Ejemplo del curso — Clase 8 · Cascada de Rentabilidad
Casa Palma · Dashboard KPI · Total anual
=SUMA(B25:M25)
En el Dashboard KPI del modelo final, cada fila representa una cuenta del P&L (Ingresos, Costo, Utilidad Bruta, etc.). La columna N suma los 12 meses (B a M) para obtener el total anual. Esta fórmula se aplica en todas las filas del P&L para construir la cascada de rentabilidad anual de Casa Palma.
Errores comunes
Números como textoSi los valores del ERP o POS vienen como texto (alineados a la izquierda), SUMA los ignora y da 0. Convertí con VALOR() o usá el truco de multiplicar por 1.

SUMAR.SI.CONJUNTO
Suma con Múltiples Filtros
Suma solo los valores que cumplen VARIOS criterios simultáneamente. La fórmula más poderosa del curso.
Sintaxis
=SUMAR.SI.CONJUNTO(rango_suma, rango_crit1, criterio1, [rango_crit2, criterio2...])
// En inglés: =SUMIFS(...) · Hasta 127 pares de rango-criterio
Cómo funciona
Solo suma las filas de rango_suma donde TODOS los criterios se cumplen al mismo tiempo. En el modelo de Casa Palma, la TablaMaestra tiene miles de filas con datos de las 3 sucursales, 12 meses, ERP+POS+Presupuesto. Con SUMIFS se puede aislar exactamente: "ingresos de la sucursal Norte, mes de marzo, fuente ERP, tipo Real".
Ejemplo del curso — Clase 6 · Dashboard con Tablas Dinámicas
Casa Palma · Dashboard KPI · Ingresos por sucursal y fecha
=SUMAR.SI.CONJUNTO(TablaMaestra[Monto_Ajustado], TablaMaestra[Rubro], "Ingresos", TablaMaestra[Sucursal], B$5, TablaMaestra[Tipo], "Real", TablaMaestra[Fuente], "ERP", TablaMaestra[Fecha_Mes], ">="&FECHA(2024,1,1), TablaMaestra[Fecha_Mes], "<="&FECHA(2024,1,31))
Esta fórmula extrae los ingresos reales del ERP de una sucursal específica (referenciada en B$5) durante enero 2024. El símbolo $ ancla la fila 5 para que al copiar la fórmula hacia la derecha cambie la columna de sucursal pero no la fila. Es la base de todo el Dashboard KPI del curso.
Errores comunes
Rango desalineadoTodos los rangos (rango_suma y rango_criterio) deben tener exactamente el mismo número de filas. Si tienen tamaños distintos devuelve #VALOR!.
Criterio de fechaPara filtrar por fechas hay que usar ">="&FECHA(2024,1,1) con el ampersand. Poner la fecha directamente como texto no funciona.

REDONDEAR
Redondeo Controlado
Redondea un número a una cantidad específica de decimales.
Sintaxis
=REDONDEAR(número, núm_decimales)
// En inglés: =ROUND(...) · 2 = centavos, 0 = entero, -3 = miles
Cómo funciona
Redondea el número al número de posiciones decimales indicado. Con 2 redondea a centavos, con 0 a entero, con -3 a miles. En modelos financieros es crítico para verificar que los porcentajes de participación sumen exactamente 100% — errores de punto flotante pueden hacer que 33.33% + 33.33% + 33.34% ≠ 100% sin redondeo.
Ejemplo del curso — Clase 14 · Supuestos de Proyección
Casa Palma · Supuestos · Validación de participación de sucursales
=SI(REDONDEAR(F15+F16+F17, 2)=1, "Sí", "No")
En la hoja de Supuestos, las tres sucursales de Casa Palma tienen un % de participación en ventas. Esta celda verifica que los tres porcentajes sumen exactamente 100% (=1 en decimal). Sin REDONDEAR, errores de punto flotante podrían hacer que 0.4+0.35+0.25 = 0.9999999 y la validación fallaría aunque los números sean correctos.
Errores comunes
Redondear vs. formatearDar formato de 2 decimales en Excel NO redondea el valor real — solo lo oculta. Si necesitás que el cálculo use el valor redondeado, debés usar REDONDEAR() explícitamente.

ABS
Valor Absoluto
Convierte cualquier número negativo en positivo, sin afectar los positivos.
Sintaxis
=ABS(número)
// ABS(-500) = 500 · ABS(500) = 500
Cómo funciona
Devuelve el valor absoluto (magnitud sin signo) de un número. En finanzas, muchas cuentas como Costos o Gastos se registran con signo negativo en el ERP o P&L. ABS permite usar esos valores en cálculos de ratios o de días sin que el signo contamine el resultado.
Ejemplo del curso — Clase 17 · Balance Proyectado
Casa Palma · Balance Proyectado · Cálculo de días de inventario
=ABS(PL_Proyectado!B8)/30 * Supuestos!$I$41
El Costo de Ventas en el P&L proyectado es negativo (sale plata). Para calcular el inventario promedio en el Balance (Costo Mensual × Días de Inventario) se necesita el valor en positivo. ABS convierte -$120,000 en $120,000 para que la fórmula del Balance funcione correctamente. Los días de inventario vienen de la celda fija de Supuestos.
Errores comunes
Ocultar problemasSi ABS se aplica a un ratio como Deuda/EBITDA donde EBITDA puede ser negativo, el resultado puede dar positivo y parecer "verde" cuando en realidad la empresa tiene EBITDA negativo — un riesgo real.

MAX
Valor Máximo / Piso en Cero
Devuelve el valor más alto de un rango. En finanzas se usa para evitar valores negativos en impuestos.
Sintaxis
=MAX(valor1, [valor2], [...])
// Truco financiero: =MAX(resultado, 0) asegura que nunca sea negativo
Cómo funciona
Devuelve el mayor valor entre los argumentos. El truco financiero clave: MAX(cálculo, 0) actúa como un "piso en cero" — si el resultado es negativo devuelve 0, si es positivo devuelve el resultado. Imprescindible para el cálculo de impuestos: una empresa no puede pagar impuesto negativo si tiene pérdida.
Ejemplo del curso — Clase 15 · P&L Proyectado
Casa Palma · P&L Proyectado · Cálculo de impuesto de renta
=MAX((B17-B15-B18), 0) * Supuestos!$I$38
Calcula el impuesto de renta proyectado de Casa Palma: (EBIT - Amortización - Intereses) × tasa de impuesto. Si la base imponible es negativa (la empresa tuvo pérdida ese mes), MAX la convierte en 0, por lo que el impuesto es $0. Sin MAX, en meses de pérdida el modelo generaría un impuesto negativo, lo cual es imposible en la realidad.
Errores comunes
No usar MAX en impuestosSin MAX(base, 0), en meses de pérdida el impuesto proyectado sería negativo, inflando artificialmente la utilidad neta y haciendo el modelo optimista de forma incorrecta.

MIN
Valor Mínimo
Devuelve el valor más bajo de un rango. En auditoría detecta el mes más crítico.
Sintaxis
=MIN(rango1, [rango2...])
// Complemento de MAX para análisis de rangos extremos
Cómo funciona
Devuelve el menor valor del rango. En auditoría financiera se usa para detectar el mes más crítico del año (ej: el mes con menores ingresos o el mayor gasto). Combinado con MAX permite calcular la volatilidad de una variable: si la diferencia entre MAX y MIN es muy grande, la variable tiene alta variabilidad que debe explicarse.
Ejemplo del curso — Clase 19 · Auditoría del Modelo
Casa Palma · Hoja Checks · Detección de valores mínimos y volatilidad
=MIN(PL_Proyectado!B4:M4) → mes con ingresos más bajos
=MAX(PL_Proyectado!B7:M7) - MIN(PL_Proyectado!B7:M7) → volatilidad del EBITDA
La hoja de auditoría verifica que los ingresos proyectados nunca caigan por debajo de un umbral. La segunda fórmula mide la diferencia entre el mes de mayor y menor EBITDA — si esa diferencia es muy grande comparada con el promedio, el modelo tiene una variabilidad inusual que debe revisarse antes de presentar.
Errores comunes
Ceros en el rangoSi hay meses futuros con 0 (aún sin proyectar), MIN devolverá 0 en vez del mínimo real. Hay que limitar el rango a los meses con datos o usar MIN con SUMAR.SI.CONJUNTO.

VNA / NPV
Valor Presente Neto
Calcula cuánto valen hoy los flujos de caja futuros de una inversión, descontados a una tasa.
Sintaxis
=VNA(tasa, flujo1, [flujo2...]) + inversión_inicial
// En inglés: =NPV(...) · La inversión inicial (negativa) se suma FUERA de VNA
Cómo funciona
Descuenta cada flujo de caja futuro a valor presente usando la tasa (normalmente el WACC o tasa de descuento del negocio). Si el VPN es positivo, la inversión crea valor; si es negativo, destruye valor. Trampa importante: VNA en Excel asume que los flujos empiezan en el período 1, por eso la inversión inicial del período 0 se suma fuera de la función.
Ejemplo del curso — Clase 18 · VPN y TIR con Flujos Reales
Casa Palma · Evaluación de Inversión Sucursal 4
=VNA(B17, C59:G59) + B59
Evalúa si abrir la cuarta sucursal de Casa Palma es rentable. B17 es el WACC (tasa de descuento). C59:G59 son los flujos de caja libre proyectados para los años 1 al 5. B59 es la inversión inicial en el año 0 (negativa, ej: -$500,000). Si el resultado es positivo, la sucursal crea valor para los accionistas a esa tasa de descuento.
Errores comunes
Incluir año 0 en VNAEl error más frecuente: poner la inversión inicial dentro del rango de VNA. VNA asume que el primer valor ocurre en el período 1, no 0. Si ponés B59 dentro del rango, la inversión se descuenta un período extra y el VPN es incorrecto.
Tasa en porcentajeLa tasa debe ser decimal (0.12 para 12%), no 12. Si ponés 12 en vez de 0.12, el resultado es absurdamente incorrecto.
📚 Clase 18 · VPN y TIR
TIR / IRR
Tasa Interna de Retorno
Calcula la tasa de descuento que hace que el VPN de una inversión sea exactamente cero.
Sintaxis
=TIR(rango_flujos, [estimación])
// En inglés: =IRR(...) · El rango DEBE incluir al menos un valor negativo (la inversión)
Cómo funciona
Encuentra iterativamente la tasa que iguala el valor presente de los flujos positivos con el de los negativos (VPN = 0). La regla de decisión: si TIR > WACC (o costo del capital), la inversión es rentable y se acepta. Si TIR < WACC, destruye valor y se rechaza. Siempre se usa junto con VPN — la TIR complementa pero no reemplaza el VPN.
Ejemplo del curso — Clase 18 · VPN y TIR con Flujos Reales
Casa Palma · Evaluación de Inversión Sucursal 4
=TIR(B59:G59) =SI(B64>B17, "✅ IRR ("&TEXTO(B64,"0.0%")&") > WACC ("&TEXTO(B17,"0.0%")&")", "❌ IRR < WACC")
B59 es el flujo año 0 (inversión negativa). B64 muestra la TIR calculada. B17 es el WACC de Casa Palma. La segunda fórmula combina TIR + TEXTO + SI para mostrar automáticamente si la inversión es viable: "✅ IRR (24.3%) > WACC (15.0%)" — el directivo ve en un vistazo si la sucursal 4 vale la pena.
Errores comunes
#NUM!Ocurre cuando los flujos son todos positivos (sin inversión inicial negativa) o cuando hay múltiples cambios de signo complejos. Verificá que el primer flujo sea la inversión en negativo.
TIR vs. TIRMTIR asume que los flujos intermedios se reinvierten a la misma TIR, lo cual es poco realista. TIRM (MIRR) permite especificar tasas de reinversión distintas — más conservadora y real para proyectos de largo plazo.

IZQUIERDA
Extraer desde la Izquierda
Extrae los primeros N caracteres de una cadena de texto, empezando por la izquierda.
Sintaxis
=IZQUIERDA(texto, núm_caracteres)
// En inglés: =LEFT(...) · IZQUIERDA("4135", 1) = "4"
Cómo funciona
Devuelve los primeros núm_caracteres del texto, contando desde la izquierda. En el diccionario financiero es clave porque los códigos contables colombianos tienen una estructura donde el primer dígito indica el tipo de cuenta: 1=Activo, 2=Pasivo, 3=Patrimonio, 4=Ingresos, 5=Costos, 6-8=Gastos. IZQUIERDA extrae ese primer dígito para clasificar automáticamente.
Ejemplo del curso — Clase 3 · Diccionario Financiero
Casa Palma · Ejercicio Diccionario · Extraer primer dígito de cuenta
=IZQUIERDA(A8, 1)
La columna A tiene los códigos de cuenta contable (ej: "4135202"). IZQUIERDA extrae "4" — ese único dígito le dice a Excel que es una cuenta de Ingresos, y con un SI()+O() anidado se asigna automáticamente el rubro, subrubro y estado financiero correspondiente sin revisar cada cuenta manualmente.
Errores comunes
Código como númeroSi el código de cuenta está guardado como número en Excel, IZQUIERDA puede no funcionar. Convertilo a texto con TEXTO(A8,"0") antes de aplicar IZQUIERDA.

DERECHA
Extraer desde la Derecha
Extrae los últimos N caracteres de una cadena de texto, empezando por la derecha.
Sintaxis
=DERECHA(texto, núm_caracteres)
// En inglés: =RIGHT(...) · DERECHA("410099", 2) = "99"
Cómo funciona
Devuelve los últimos núm_caracteres del texto. En los planes de cuentas contables colombianos, las cuentas que terminan en "99" son cuentas de subtotal o resumen — no deben incluirse en el cálculo porque ya incluyen el total de las subcuentas. DERECHA permite identificarlas y excluirlas automáticamente.
Ejemplo del curso — Clase 3 · Diccionario Financiero
Casa Palma · Ejercicio Diccionario · Detección de cuentas subtotal
=SI(DERECHA(A8,2)="99", "Subtotal", SI(C8="4", "Ingresos", SI(C8="5", "Costo de Ventas", ...)))
Primero verifica si la cuenta termina en "99". Si sí, la clasifica como "Subtotal" y no se suma en el P&L (evita doble conteo). Si no termina en "99", continúa con la lógica de clasificación por primer dígito. Esta verificación es el primer paso del árbol de clasificación del diccionario de Casa Palma.
Errores comunes
Mismo que IZQUIERDASi el código es número, los dos últimos dígitos de "4135" como número son "35" — pero DERECHA("4135", 2) también da "35". El problema surge si el código tiene ceros al inicio (ej: "0410") que Excel puede eliminar si lo trata como número.

TEXTO
Número a Texto con Formato
Convierte un número en texto con el formato visual que especifiques, para concatenar en etiquetas dinámicas.
Sintaxis
=TEXTO(valor, formato)
// Formatos comunes: "0.0%" · "$#,##0" · "0.0x" · "dd/mm/yyyy"
Cómo funciona
Convierte el valor numérico en texto aplicando el formato que le indicás (el mismo sistema de formatos de celdas de Excel). Permite construir etiquetas dinámicas que mezclan texto fijo con valores calculados, como títulos de gráficos que se actualizan solos o celdas de diagnóstico con íconos y números.
Ejemplo del curso — Clase 18 · Evaluación de Inversión
Casa Palma · Evaluación Sucursal 4 · Diagnóstico automático TIR vs WACC
=SI(B64>B17, "✅ IRR ("&TEXTO(B64,"0.0%")&") > WACC ("&TEXTO(B17,"0.0%")&")", "❌ IRR < WACC")
Combina SI + TEXTO + & (concatenación). Si la TIR (B64=0.243) supera el WACC (B17=0.15), muestra "✅ IRR (24.3%) > WACC (15.0%)". Sin TEXTO, concatenar 0.243 con texto daría "✅ IRR (0.243) > WACC (0.15)" — ilegible. TEXTO formatea el número como porcentaje dentro del texto para que el mensaje sea claro para el directivo.
Errores comunes
Resultado es textoEl resultado de TEXTO es texto, no número. No puede usarse en cálculos matemáticos posteriores. Solo usarlo para etiquetas y mensajes, nunca como insumo de otra fórmula numérica.

FECHA
Construir Fechas para Filtros
Crea una fecha a partir de año, mes y día. Imprescindible para filtrar por períodos en SUMIFS.
Sintaxis
=FECHA(año, mes, día)
// En inglés: =DATE(...) · FECHA(2024, 1, 1) = 01/01/2024
Cómo funciona
Crea una fecha válida de Excel (número serial) a partir de sus componentes. En modelos financieros se usa como criterio en SUMIFS para filtrar datos de un período específico: ">="&FECHA(2024,1,1) selecciona todos los registros desde el 1° de enero de 2024. Los componentes pueden ser celdas — si el año está en B1 y el mes en B2, podés construir fechas dinámicas.
Ejemplo del curso — Clase 22 · Dashboard KPI
Casa Palma · Dashboard KPI · Filtro por mes con SUMIFS
=SUMAR.SI.CONJUNTO(TablaMaestra[Monto_Ajustado], TablaMaestra[Rubro], "Ingresos", TablaMaestra[Fecha_Mes], ">="&FECHA(2024,1,1), TablaMaestra[Fecha_Mes], "<="&FECHA(2024,1,31))
Filtra solo los registros de enero 2024 de la TablaMaestra. FECHA construye las fechas de inicio (1° de enero) y fin (31 de enero) del mes como valores que SUMIFS puede comparar. Si las fechas se pusieran como texto ("01/01/2024"), la comparación fallaría porque Excel no puede comparar texto con fechas reales.
Errores comunes
Fecha como textoEl error más frecuente en SUMIFS: escribir la fecha directamente como texto en el criterio. =SUMAR.SI.CONJUNTO(..., ">="&"01/01/2024") no funciona. Siempre usá FECHA() o una referencia a celda con fecha.

AÑO
Extraer el Año de una Fecha
Extrae el año de una fecha como número. Útil para crear columnas auxiliares de clasificación temporal.
Sintaxis
=AÑO(fecha)
// En inglés: =YEAR(...) · AÑO("15/03/2024") = 2024
Cómo funciona
Devuelve el componente "año" de una fecha como número entero. En el modelo de Casa Palma se usa para crear una columna auxiliar "Año" en la tabla de ventas diarias del POS, que luego sirve para segmentar los datos en tablas dinámicas por año sin necesidad de filtros manuales.
Ejemplo del curso — Clase 21 · Tablas Dinámicas para Dashboard
Casa Palma · Ventas Diarias (POS) · Columna auxiliar Año
=AÑO(Ventas_Diarias[[#Esta Fila],[Fecha]])
La tabla de ventas del POS tiene fechas individuales de cada transacción. Esta columna calculada extrae el año de cada fecha (2023, 2024, etc.) y se agrega automáticamente a todos los registros de la tabla. Cuando se crea la tabla dinámica para el dashboard, el campo "Año" permite filtrar y segmentar ventas por año con un clic.
Errores comunes
Fecha como textoSi las fechas del POS vienen como texto (frecuente cuando se exporta mal), AÑO devuelve #VALOR!. Primero convertir con FECHANUMERO() o Power Query.

PROMEDIO
Promedio de Valores
Calcula la media aritmética de un rango. En auditoría detecta valores atípicos comparando contra el promedio.
Sintaxis
=PROMEDIO(rango1, [rango2...])
// En inglés: =AVERAGE(...) · Ignora celdas vacías pero incluye ceros
Cómo funciona
Suma todos los valores del rango y los divide por la cantidad de celdas con números. En el modelo de Casa Palma se usa en la hoja de Auditoría para comparar un mes específico contra el promedio de los demás — si un mes tiene un valor muy alejado del promedio, puede ser un error de entrada o un evento real que debe explicarse.
Ejemplo del curso — Clase 19 · Auditoría del Modelo
Casa Palma · Hoja Checks · Detección de outliers en EBITDA
=ABS(PL_Proyectado!H10 - PROMEDIO(PL_Proyectado!B10:G10, PL_Proyectado!I10:M10))
Mide qué tan lejos está el EBITDA de agosto (columna H) del promedio de los demás 11 meses. Si la diferencia es mayor a un umbral (ej: 20% del promedio), la celda de auditoría muestra alerta. Se excluye agosto del promedio usando dos rangos separados — si se incluyera, el outlier inflaría el promedio y reduciría la sensibilidad de la detección.
Errores comunes
Ceros vs. vacíosPROMEDIO incluye celdas con 0 en el denominador pero ignora vacías. Si los meses futuros tienen 0 en vez de estar vacíos, el promedio queda distorsionado hacia abajo. Usá PROMEDIO.SI para excluir ceros si es necesario.

CONTAR.SI
Contar con Condición
Cuenta cuántas celdas de un rango cumplen un criterio específico.
Sintaxis
=CONTAR.SI(rango, criterio)
// En inglés: =COUNTIF(...) · Criterio puede ser texto, número o comparación
Cómo funciona
Recorre el rango y cuenta cuántas celdas coinciden con el criterio. En el sistema de auditoría del modelo de Casa Palma, se usa para contar cuántas celdas de verificación contienen la palabra "ERROR". Si la cuenta es mayor a 0, el modelo tiene problemas.
Ejemplo del curso — Clase 19 · Auditoría del Modelo
Casa Palma · Hoja Checks · Contador de errores por bloque
=SI(CONTAR.SI(C5:C7, "OK")=3, "MODELO LIMPIO ✓", "HAY ERRORES")
La hoja de Checks tiene celdas que devuelven "OK" o "ERROR" según si el P&L, Balance y Flujo de Caja cuadran correctamente. CONTAR.SI cuenta cuántas de esas 3 celdas dicen "OK". Si son exactamente 3 (todas OK), el modelo está limpio. Si alguna dice "ERROR", el SI muestra la alerta. Este es el sistema de semáforo de auditoría del modelo final.
Errores comunes
Sensible a mayúsculas en conceptoCONTAR.SI es insensible a mayúsculas: "error", "ERROR" y "Error" se cuentan igual. Eso está bien para auditoría, pero verificá que las celdas fuente usen consistentemente el mismo texto.
